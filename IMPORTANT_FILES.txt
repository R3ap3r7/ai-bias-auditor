AI Bias Auditor - Important Files and Run Notes
===============================================

GitHub repository:
https://github.com/R3ap3r7/ai-bias-auditor

Do not paste or commit real API keys into source files. Put your Gemini key in a local .env file:

GEMINI_API_KEY=your-rotated-google-ai-studio-key
GEMINI_MODEL=gemini-2.5-flash

Because an API key was shared in chat, rotate it in Google AI Studio before using it again.
Gemini is used only for the final plain-English analysis report. Model selection and hyperparameter tuning run locally.

Local Run
---------
1. uv venv --python 3.12
2. source .venv/bin/activate
3. uv pip install -r requirements.txt
4. cp .env.example .env
5. Edit .env and add your Gemini key.
6. python scripts/download_demos.py
7. uvicorn app.main:app --reload
8. Open the Flutter UI at http://localhost:5050 after starting it in step 10.
9. For an external model audit, use prediction CSV mode. Uploaded pickle/joblib model artifacts are disabled by default.
10. For the Flutter/Firebase UI, start the backend, then run:
    cd flutter_firebase_auditor
    flutter run -d chrome --web-port 5050 --dart-define=API_BASE_URL=http://127.0.0.1:8000

Validation
----------
pytest -q
python -m compileall app scripts tests

Key Files
---------
app/main.py
FastAPI app entry point. Defines the health check, demo dataset API, CSV upload API, prediction CSV API, audit API, and PDF report download API. User-facing routes redirect to Flutter.

app/audit.py
Core bias audit engine. Handles CSV cleaning, missing values, outcome normalization, data-only pre-audit, representation balance, proxy detection, local hyperparameter tuning, tuned model comparison, uploaded-model prediction audits, Fairlearn metrics, conditional same-background analysis, intersectional analysis, row-level decision audit traces, run traceability metadata, feature importance, mitigation simulation, severity scoring, and Gemini report generation.

app/demo_data.py
Demo dataset registry and loader for COMPAS, UCI Adult, and German Credit. Requires the downloaded CSVs in data/demos and does not generate synthetic fallback data.

app/report.py
Builds downloadable PDF reports with ReportLab, including model comparison, traceability, conditional fairness, intersectional bias, and decision audit trace sections.

scripts/download_demos.py
Downloads and preprocesses COMPAS, UCI Adult, and German Credit demo CSVs into data/demos.

tests/test_audit.py
Regression tests for outcome normalization, representation/proxy/bias detection, tuned model comparison, row-level audit traces, traceability metadata, intersectional analysis, supported model pipelines, PDF generation, severity scoring, pre-audit API flow, uploaded model API flow, prediction validation, and upload/audit API flow.

requirements.txt
Runtime and test dependencies.

pyproject.toml
Project metadata and pytest configuration.

Dockerfile
Cloud Run compatible container definition.

flutter_firebase_auditor/lib/screens/dashboard_screen.dart
Flutter web UI for guest mode, Google login, audit workspace, separated results review, saved Firebase history, prediction CSV controls, pre-audit, post-audit, model comparison, governance trace, feature importance, mitigation simulation, PDF reports, and Gemini/local analysis.

flutter_firebase_auditor/lib/services/backend_client.dart
Typed Flutter HTTP client for the FastAPI upload, demo, prediction CSV, pre-audit, full audit, and PDF report APIs.

flutter_firebase_auditor/lib/services/audit_repository.dart
Firebase Auth and Firestore repository. Handles Google sign-in, user profile upserts, audit summary saves, and trace preview saves.

flutter_firebase_auditor/lib/models/audit_record.dart
Firestore audit history model used by the Flutter saved-history and recent-run UI.

flutter_firebase_auditor/lib/theme/app_theme.dart
Material 3 theme and governance palette.

flutter_firebase_auditor/lib/widgets/ui_shell.dart
Reusable panels, section headers, metric tiles, status pills, and empty states for the Flutter UI.

flutter_firebase_auditor/firebase.json
Firebase Hosting, Firestore rules, indexes, and Flutter Firebase project configuration.

Known Notes
-----------
Gemini is optional at runtime. If GEMINI_API_KEY is missing or Gemini fails, the app generates a deterministic local analysis report instead of showing a setup/config message.
Gemini is not used for choosing models. "Compare All Tuned Models" runs local GridSearchCV searches and ranks candidates by balanced accuracy penalized by the largest demographic parity and equalized odds gaps.
The decision audit trace uses local baseline perturbation, not full SHAP. It gives per-row contribution evidence for the MVP without adding a heavy dependency or limiting support to one model family.
The UI uses Tailwind CDN for MVP speed. For production hardening, replace it with a compiled Tailwind build.
The Flutter/Firebase UI is the only supported frontend. FastAPI is the API/report backend and redirects user-facing routes to Flutter through FRONTEND_URL.
Uploaded pickle/joblib models can execute code when loaded and are disabled by default. Use prediction CSV mode for external model outputs.
Cloud Run deployment is currently blocked until billing is enabled on Google Cloud project ai-bias-auditor-2604171603. Firebase Hosting alone cannot run the Python audit engine.
