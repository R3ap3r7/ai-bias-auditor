# Stitch Design Reference

Stitch project: `projects/10737912738132937207`

Generated screen: `projects/10737912738132937207/screens/ee80c2054741437988b30a199bcf801a`

The Flutter implementation follows the generated dashboard direction:

- Left audit-history rail backed by Firebase.
- One scrollable governance workspace, not a marketing landing page.
- Dataset setup, protected-attribute mapping, outcome selection, model source selection, and run controls first.
- Separate Pre-model audit, Post-model audit, Audit trace, and Gemini analysis sections.
- Material 3 inspired tonal surface hierarchy.
- Radius capped at 8px for panels and buttons.
- Semantic blue, green, warning, and red usage only.
- No decorative blobs, gradients, purple theme, beige/tan palette, or card-in-card layout.

- Background: `#F8F9FA`
- Primary: `#005BC0`
- Success: `#006E2B`
- Error: `#9F403D`
- Surface low: `#F1F4F6`
- Surface high: `#E3E9EC`
